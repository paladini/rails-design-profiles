Design Infrastructure for Rails

Change the system, not the whole site.

Use design references to guide decisions. Use a small, versioned token profile to make those decisions real in your Rails application.

Read the Installation GuideSee the Workflow ↓

A Deliberate Pipeline

01Reference a visual direction
02Review the tokens that belong to your product
03Activate a profile your components already understand

No copied CSS. No black-box theme compiler. Just an explicit contract between design intent & production UI.

The Principle

DESIGN.md gives an agent context. CSS variables give a team control.

The Workflow

STEP / 01

Reference

Bring an openly licensed DESIGN.md into the project, or add one your team owns.

STEP / 02

Decide

Translate only the useful visual decisions into readable YAML tokens that can be discussed in a pull request.

STEP / 03

Apply

Activate the profile. The gem renders standard --rdp-* variables for your Rails components.

The Production Contract

Small file. Visible consequences.

Your team owns this file. Your components choose how to consume it. That is the boundary.

# config/design_profiles.yml
active: harbor
profiles:
  harbor:
    reference: config/design_profiles/references/harbor.md
    tokens:
      color-primary: "#1455a5"
      color-surface: "#eef4f5"
      color-text: "#172133"
      font-body: "Public Sans, sans-serif"
