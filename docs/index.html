A useful boundary

References guide choices. Tokens make those choices dependable in production.

The workflow

A focused path from design direction to Rails CSS variables.

01 / Reference

Bring context

Install an openly licensed reference or add one your team already maintains.

02 / Decide

Choose tokens

Turn the decisions worth shipping into explicit YAML values.

03 / Apply

Use the profile

The gem emits --rdp-* variables. Your components stay in control.

What makes it different

It is not a theme generator.

The gem does not copy another site, replace your CSS, or turn design prose into a black box.

✓Use a reference to give an agent design context.
✓Review every production token in a normal pull request.
✓Switch profiles without handing away ownership of the interface.

Make visual change feel native to your Rails workflow.
